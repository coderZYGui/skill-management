# skill-management 项目知识总入口

## 作用

`AGENTS.md` 是整个仓库的总索引，目标是让人或 AI 在最短时间内理解这个项目的定位、当前实现边界，以及 `docs/knowledge` 中每一组文档该从哪里读起。

## 当前项目快照

- 项目名称：`skill-management`
- 项目类型：基于 Spring Boot 的单模块 Web 应用
- 主要技术：Java 17、Spring Boot 3.5.13、Spring MVC、Maven、Lombok、原生 HTML/CSS/JavaScript
- 启动入口：`src/main/java/com/xun/TestDemoApplication.java`
- 核心业务对象：`Skill`
- 核心能力：
  - 提供 `Skill` 的新增、查询、更新、删除 REST API
  - 提供静态管理页面 `src/main/resources/static/index.html`
  - 提供预置 Skill 模板快速填充
  - 统一使用 `Result<T>` 作为响应包装
- 当前实现边界：
  - 数据仅保存在内存 `ConcurrentHashMap`，服务重启后会丢失
  - 当前没有数据库、Repository、鉴权、权限控制、文件存储、消息队列
  - 测试与构建依赖本机正确配置 `JAVA_HOME`
  - 页面中文文案和部分日志文本存在编码异常，属于当前已知问题

## 建议阅读顺序

1. 先看 `docs/knowledge/01-架构设计`，快速理解当前工程结构、层次职责和模块依赖。
2. 再看 `docs/knowledge/02-模块详解`，定位每个类、页面和响应对象的具体职责。
3. 如果要本地运行或继续开发，再看 `docs/knowledge/03-开发指南`。
4. 如果遇到启动、构建、接口或仓库问题，再看 `docs/knowledge/04-问题排查`。
5. 如果要统一术语、技术边界和实现方式，再看 `docs/knowledge/05-规范参考`。
6. 如果要确认第三方依赖和外部模块，再看 `docs/knowledge/06-外部依赖`。
