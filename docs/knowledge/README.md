# knowledge 知识库说明

## 目录定位

`docs/knowledge` 是 `skill-management` 项目的项目知识库，既面向开发者，也面向需要快速理解仓库的 AI。这里的内容以“当前代码真实实现”为中心，不写脱离项目实际的泛化说明。

## 当前项目概况

- 这是一个 Spring Boot 单体 Web 应用。
- 后端提供 `Skill` 的 CRUD API。
- 前端通过静态页面调用 `/api/skills` 完成管理操作。
- 数据当前仅保存在内存，不做持久化。

## 子目录说明

- `01-架构设计`：回答“这个项目是怎么组织起来的”。
- `02-模块详解`：回答“每个类、页面、对象到底做什么”。
- `03-开发指南`：回答“如何运行、如何扩展、如何继续开发”。
- `04-问题排查`：回答“启动失败、接口失败、环境异常时怎么查”。
- `05-规范参考`：回答“术语怎么定义、技术边界是什么、当前异步策略如何理解”。
- `06-外部依赖`：回答“项目依赖了哪些外部库、插件和运行时能力”。

## 使用建议

1. 新人第一次接触仓库，先读 `01-架构设计`。
2. 需要改代码时，再读 `02-模块详解` 和 `03-开发指南`。
3. 遇到异常时，先查 `04-问题排查`。
4. 需要统一概念和边界时，再查 `05-规范参考` 与 `06-外部依赖`。
