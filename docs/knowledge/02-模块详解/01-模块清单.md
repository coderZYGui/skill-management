## 4. 接口清单

当前后端对外暴露的接口如下：

| 方法 | 路径 | 作用 |
| --- | --- | --- |
| `GET` | `/api/skills` | 查询全部 Skill |
| `GET` | `/api/skills/{id}` | 按 ID 查询单个 Skill |
| `POST` | `/api/skills` | 新增 Skill |
| `PUT` | `/api/skills/{id}` | 更新指定 Skill |
| `DELETE` | `/api/skills/{id}` | 删除指定 Skill |

## 5. `Skill` 对象字段

`Skill` 当前包含以下字段：

| 字段 | 含义 |
| --- | --- |
| `id` | 主键 ID，由服务层自动分配 |
| `name` | 唯一标识名，前端要求使用小写字母、数字和连字符 |
| `displayName` | 页面展示名称 |
| `description` | Skill 描述 |
| `content` | Skill 指令内容，通常为 Markdown |
| `author` | 作者 |
| `category` | 分类，如 `code-review`、`document` |
| `createdAt` | 创建时间 |
| `updatedAt` | 更新时间 |

## 6. 页面功能清单

`index.html` 当前实现了以下页面功能：

- 表单录入 Skill 基本信息
- 校验 `name` 格式和 `description` 必填
- 列表展示所有 Skill
- 查看单个 Skill 详情弹窗
- 编辑已有 Skill
- 删除 Skill
- 提供预置 Skill 模板菜单
- 使用 toast 提示操作结果

## 7. 当前模块层面的已知问题

- `SkillService` 同时承担业务逻辑和存储职责，耦合偏高。
- 页面中的中文文本出现乱码，说明前端源码编码需要统一。
- 日志中的部分中文也存在乱码现象。
- 测试类只覆盖容器启动，没有覆盖 CRUD 逻辑。
