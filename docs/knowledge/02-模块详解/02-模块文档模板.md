# 模块文档模板

## 1. 用途

当后续项目继续新增模块、类或页面时，建议按照这份模板补充对应文档。模板目标不是追求形式完整，而是保证别人能快速回答这几个问题：

- 这个模块在哪
- 它负责什么
- 它依赖谁
- 它向外暴露什么
- 它有哪些已知限制

## 3. 结合当前项目的填写示例

下面用 `SkillService` 做一个真实示例。

### 模块名称

`SkillService`

### 基本信息

- 位置：`src/main/java/com/xun/service/SkillService.java`
- 类型：服务类
- 当前状态：已实现并投入主流程使用

### 模块职责

- 提供 Skill 的查询、新增、更新、删除逻辑
- 管理内存中的 `skillStore`
- 维护自增 ID、创建时间和更新时间

### 输入与输出

- 输入：来自 `SkillController` 的 `id` 或 `Skill` 对象
- 输出：`Skill`、`List<Skill>` 或删除结果布尔值

### 依赖关系

- 依赖 `Skill`
- 依赖 `ConcurrentHashMap`
- 被 `SkillController` 调用

### 关键流程

1. 控制器调用服务方法
2. 服务层从 Map 中读取或写入数据
3. 服务层返回处理结果给控制器

### 已知限制

- 当前没有持久化
- 当前没有事务和复杂校验
- 服务层直接持有存储实现

## 4. 建议

后续新增模块时，文档至少要补到能让阅读者不打开代码也知道“职责、输入输出、依赖和限制”的程度。
