# 01-架构设计

## 目录作用

这一组文档用于回答项目的整体结构问题，包括工程分层、模块依赖、调用关系，以及在当前单模块实现下如何理解“跨模块通信”。

## 当前项目基础信息

- 当前项目属于单模块 Spring Boot 应用，不是多模块 Maven 工程。
- 业务中心是 `Skill` 管理。
- 前后端部署在同一个应用中：后端提供 API，前端页面由 `static/index.html` 直接托管。
- 存储形态是内存存储，因此当前架构更偏演示版或原型版。
