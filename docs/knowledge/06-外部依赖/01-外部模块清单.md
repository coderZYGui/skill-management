# 外部模块清单

## 1. Maven 依赖

### 1.1 `spring-boot-starter-web`

- 来源：Spring Boot
- 用途：
  - 提供 Spring MVC
  - 提供内嵌 Web 容器
  - 提供 JSON 序列化支持
- 当前落点：
  - `@RestController`
  - `@GetMapping`、`@PostMapping`、`@PutMapping`、`@DeleteMapping`
  - 页面与 API 的 HTTP 访问能力

### 1.2 `lombok`

- 来源：Project Lombok
- 用途：
  - 生成 getter、setter
  - 生成构造器
  - 生成日志字段
- 当前落点：
  - `Skill`
  - `Result`
  - `SkillController`
  - `SkillService`

### 1.3 `spring-boot-starter-test`

- 来源：Spring Boot
- 用途：
  - 提供测试相关基础能力
  - 集成 JUnit 5 和 Spring Test
- 当前落点：
  - `TestDemoApplicationTests`

## 2. 构建插件

### 2.1 `maven-compiler-plugin`

- 用途：编译 Java 代码
- 当前额外配置：为 Lombok 设置注解处理器路径

### 2.2 `spring-boot-maven-plugin`

- 用途：
  - 支持 Spring Boot 打包与运行
  - 构建可执行 jar
- 当前额外配置：排除了 Lombok 打包

## 3. JDK 标准库能力

当前代码除了业务类，也依赖若干 JDK 内置能力：

- `ConcurrentHashMap`：内存并发安全存储
- `AtomicLong`：自增 ID 生成
- `LocalDateTime`：时间生成
- `DateTimeFormatter`：时间格式化

这些不属于第三方依赖，但属于当前业务运行时的重要基础能力。

## 4. 浏览器运行时能力

前端页面依赖浏览器原生能力：

- `fetch`：调用后端 API
- DOM API：表单读写、表格渲染、弹窗与提示控制
- `confirm`：删除确认

当前页面没有引入 React、Vue 或其他前端框架。

## 5. 当前明确没有接入的外部模块

为了避免误解，需要明确记录当前未接入以下能力：

- 数据库
- Redis
- 消息队列
- 对象存储
- 搜索引擎
- 第三方认证
- 第三方 AI 平台 SDK

## 6. 当前依赖结论

这个项目的外部依赖非常轻量，主要依靠 Spring Boot Web、Lombok 和测试依赖完成整体运行。大部分业务逻辑都还停留在本地内存和原生页面层面，因此理解项目时应优先从业务代码本身入手，而不是假设存在复杂外部基础设施。
