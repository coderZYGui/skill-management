# 术语表

## Skill

项目中的核心业务对象，表示一条可管理的 AI Skill 记录。它包含标识名、显示名、描述、内容、作者、分类和时间字段。

## `name`

Skill 的标识名。前端要求它只包含小写字母、数字和连字符，例如 `code-review`。

## `displayName`

Skill 的展示名称，主要用于页面上给用户看，不一定等同于 `name`。

## `description`

Skill 的说明文字，用于描述它做什么、什么时候使用。

## `content`

Skill 的具体内容，当前设计上通常保存 Markdown 指令文本。

## `category`

Skill 的分类。当前页面内置了以下几类：

- `code-review`
- `code-generation`
- `document`
- `deployment`
- `data-analysis`
- `other`

## `Result<T>`

统一响应包装对象，包含：

- `code`
- `message`
- `data`

后端接口当前都通过它返回结果。

## 预置模板

前端 `index.html` 中定义的一组 `PRESET_SKILLS`。用户可以点击快捷入口，把常见 Skill 模板快速填入表单。

## 内存存储

指 `SkillService` 通过 `ConcurrentHashMap<Long, Skill>` 暂存数据的方式。它的特点是实现简单、无需数据库，但应用重启后数据会全部丢失。

## CRUD

Create、Read、Update、Delete 的缩写，对应当前项目最核心的增删改查能力。

## 静态页面

指 `src/main/resources/static/index.html`。Spring Boot 启动后会直接托管这个页面，不需要额外前端打包流程。

## 上下文加载测试

指 `TestDemoApplicationTests` 中的 `contextLoads()` 测试，它只验证 Spring 容器能否成功启动，不代表业务逻辑已经被充分测试。
