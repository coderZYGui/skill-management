# 构建与运行

## 1. 环境要求

- JDK 17
- Maven 3.9 及以上
- 正确配置 `JAVA_HOME`
- Windows、macOS、Linux 均可运行，当前仓库主要以本地开发为目标

## 2. 当前仓库的已验证情况

在当前机器上执行 `mvn test` 时，出现了如下问题：

```text
The JAVA_HOME environment variable is not defined correctly
```

这说明当前代码仓库并没有确认失败，但本机环境尚未配置好 Java 运行环境。

## 3. 本地启动前检查

建议先执行以下命令：

```powershell
echo $env:JAVA_HOME
mvn -v
```

如果 `JAVA_HOME` 为空，或者 `mvn -v` 无法正常输出 Java 版本，需要先修复环境变量。

## 4. 常用命令

### 4.1 启动项目

```powershell
mvn spring-boot:run
```

### 4.2 运行测试

```powershell
mvn test
```

### 4.3 打包项目

```powershell
mvn clean package
```

## 5. 访问方式

项目成功启动后，默认可以访问：

- 页面地址：`http://localhost:8080/`
- API 地址：`http://localhost:8080/api/skills`

## 6. 接口调用示例

### 查询全部 Skill

```powershell
Invoke-RestMethod -Uri "http://localhost:8080/api/skills" -Method Get
```

### 新增 Skill

```powershell
$body = @{
  name = "code-review"
  displayName = "代码审查助手"
  description = "Review code changes"
  content = "# Code Review"
  author = "Seekseek"
  category = "code-review"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:8080/api/skills" -Method Post -Body $body -ContentType "application/json"
```

## 7. 运行时注意事项

- 数据保存在内存中，每次重启应用都会丢失。
- 页面依赖后端 API 正常返回，后端未启动时页面无法正常使用。
- 当前没有数据库初始化步骤，也没有额外配置文件要求。

## 8. IDE 运行方式

如果使用 IntelliJ IDEA，可以直接运行 `TestDemoApplication` 主类。前提仍然是本机已经配置可用的 JDK 17。
