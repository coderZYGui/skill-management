# 排查入口与信息采集

## 1. 排查的基本原则

遇到问题时，先收集事实，再判断是环境问题、代码问题、接口问题还是设计使然。当前项目规模不大，很多问题都可以通过目录、环境变量、日志和接口返回快速定位。

## 2. 建议优先收集的信息

### 2.1 Git 与目录信息

```powershell
pwd
Get-ChildItem -Force
git status
git remote -v
```

### 2.2 Java 与 Maven 环境

```powershell
echo $env:JAVA_HOME
where.exe java
mvn -v
```

### 2.3 项目运行信息

```powershell
mvn spring-boot:run
```

观察：

- 是否成功启动
- 端口是否被占用
- 控制台是否出现异常栈

### 2.4 接口可用性

```powershell
Invoke-RestMethod -Uri "http://localhost:8080/api/skills" -Method Get
```

## 3. 当前项目中已经遇到或容易遇到的问题

### 3.1 `fatal: not a git repository`

现象：

```text
fatal: not a git repository (or any of the parent directories): .git
```

常见原因：

- 命令在项目根目录之外执行

当前项目中的实际情况：

- `C:\Users\Seekseek\Desktop` 不是 Git 仓库
- `C:\Users\Seekseek\Desktop\skill-management` 才是仓库根目录

处理方法：

```powershell
cd C:\Users\Seekseek\Desktop\skill-management
git status
```

### 3.2 `JAVA_HOME` 未配置

现象：

```text
The JAVA_HOME environment variable is not defined correctly
```

原因：

- Maven 找不到可用 JDK

处理思路：

1. 确认本机已安装 JDK 17
2. 配置 `JAVA_HOME`
3. 重新打开终端后再执行 `mvn -v`

### 3.3 页面打开但操作失败

可能原因：

- 后端服务没有启动
- `/api/skills` 接口报错
- 浏览器请求被异常中断

排查步骤：

1. 先确认 `http://localhost:8080/` 能访问
2. 再确认 `GET /api/skills` 是否返回 JSON
3. 最后看浏览器控制台和后端日志

### 3.4 数据重启后消失

这不是 bug，而是当前实现方式决定的：

- `SkillService` 使用的是内存 `ConcurrentHashMap`
- 应用重启后，内存会被清空

如果要保留数据，必须接入数据库或文件持久化。

### 3.5 中文乱码

当前项目中存在两类乱码现象：

- 页面文案乱码
- 日志或注释中的中文乱码

常见原因：

- 源文件编码不统一
- 编辑器或 IDE 未使用 UTF-8

建议：

- 统一将源码与资源文件保存为 UTF-8
- 在 IDE 中确认 project encoding

### 3.6 `Skill not found`

现象：

- 访问不存在的 `id`
- 更新或删除不存在的 `id`

当前表现：

- 接口返回 `404`
- 响应体由 `Result.error(404, "Skill not found")` 生成

## 4. 快速判断问题属于哪一层

- 构建失败：优先看 Java、Maven 和 `pom.xml`
- 页面加载失败：优先看 Spring Boot 是否启动、端口是否正常
- 页面操作失败：优先看 `/api/skills` 接口和浏览器请求
- 数据不保存：优先看是否理解了“内存存储”边界
- 文案乱码：优先看文件编码而不是业务逻辑

## 5. 当前排查结论

对于这个项目，大部分问题都不是复杂业务故障，而是环境配置、请求链路或实现边界理解不一致导致的。优先确认“是不是在正确目录、Java 是否可用、服务是否启动、数据是不是本来就不持久化”，通常就能快速定位。
